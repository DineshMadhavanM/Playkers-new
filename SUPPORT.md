# Support

Thanks for your interest in contributing! Here’s how to get help and support.

## Where to ask questions

- GitHub Issues: use the "question" template for general questions.
- Bug reports: use the "bug report" template with clear steps to reproduce.
- Feature requests: use the "feature request" template and describe the use case.

## Response times

- We try to respond to new issues within 3–5 business days.
- Complex requests may take longer; PRs are always welcome.

## Security disclosures

- Please do not open public issues for security vulnerabilities.
- Follow SECURITY.md for responsible disclosure instructions.

## Before filing an issue

- Search existing issues and discussions.
- Include version info, environment details, logs, and screenshots.

## Development quickstart

- See README.md for setup, environment variables, and running locally.
- See CONTRIBUTING.md for coding standards, branching, and PR process.
